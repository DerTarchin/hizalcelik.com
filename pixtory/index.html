Pixtory

Maps all images by over 16 million colors.

category

Art, Code, UI/UX Design

date

Fall 2016

technologies used

Python, HTML, CSS, Javascript, jQuery

course

Concept Studio: Space and Time

Pixtory is a quick project that maps out all the images in a folder by color. The result allows you to nonlinearly explore your personal photos and potentially rediscover old images you've forgotten about, particulary if you have over 25,000 photos like myself.

You can read the official project report here.

Process

A python script analyzes all the images in a folder, pixel-by-pixel, assigning a single image file to each possible RGB value (over 16m total) along with the coordinates the pixel was located. This data is exported as a JSON objects.

A single-page Javascript app then loads the JSON objects and allows the user to change the color and see an image that contains it. Users can adjust color based on RGB values or a colorpicker. On hover, a mask appears showing the pixel location of that color.

Challenges

Speed was the biggest challenge, and always has room for more improvement. As of now, the python script can take days to analyze 20,000 DSLR-quality photos. The webapp used to take over 20 minutes to load the JSON files (due to JSON filesize), but with a combination in compressed text in JSON objects and improving loading code over many iterations, the load time is now closer to 10 seconds with minimal render freezing.

Another issue was getting images to appear quickly when dragging the colorpicker. Creating a copy of all images at a max-size of 1000x1000 pixels allowed for quicker loading by the browser. I developed code that would scale the color-pixel location of the original dimensions to the browser-size version as well. However, I have plans to improve this code much more, see below.

I plan to revisit this project and add the following improvements: ability to recursively find images starting at root folder, automatically resize and compress images to two new versions (1000px and 100px), improve python analysis speed (use multithreading?), improve app speed and efficiency. I plan on incorporating lazy loading from 100px to 1000px versions to improve speed when scrubbing with the colorpicker, as well as writing more efficient code for changing the main picture during color changes. If I have time, I may package it to be available to anyone. Possibly compressing JSON files with GZIP and minimizing actual text in the files may improve load speed as well.
